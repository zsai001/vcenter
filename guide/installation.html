安装部署 ​

本文档提供 VCenter 的详细安装部署指南。

系统要求 ​

硬件配置 ​

规模CPU内存磁盘
小型（< 10 节点）2 核4 GB40 GB
中型（10-50 节点）4 核8 GB100 GB
大型（> 50 节点）8 核16 GB200 GB

操作系统 ​

  • Ubuntu 20.04 / 22.04 LTS（推荐）
  • CentOS 7 / 8 / Stream
  • Debian 11 / 12
  • RHEL 8 / 9

依赖软件 ​

  • Docker 20.10+ 和 Docker Compose 2.0+（Docker 部署）
  • Go 1.24+（源码编译）
  • PostgreSQL 14+（生产环境推荐）

Docker 部署（推荐） ​

1. 安装 Docker ​

bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | bash

# CentOS
yum install -y docker-ce docker-ce-cli containerd.io
systemctl enable docker && systemctl start docker

2. 创建配置目录 ​

bash
mkdir -p /opt/vcenter/{data,config}
cd /opt/vcenter

3. 创建 docker-compose.yml ​

yaml
version: '3.8'

services:
  vcenter:
    image: vcenter/server:latest
    container_name: vcenter
    restart: unless-stopped
    ports:
      - "8090:8090"
    volumes:
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
      - VCENTER_MODE=release
      - VCENTER_JWT_SECRET=${JWT_SECRET}

4. 创建环境变量 ​

bash
echo "JWT_SECRET=$(openssl rand -hex 32)" > .env

5. 启动服务 ​

bash
docker compose up -d

6. 查看日志 ​

bash
docker compose logs -f

二进制部署 ​

1. 下载二进制 ​

bash
# 从 GitHub Release 下载
curl -LO https://github.com/zsai001/vcenter/releases/latest/download/vcenter-linux-amd64.tar.gz

# 解压
tar -xzf vcenter-linux-amd64.tar.gz
cd vcenter

2. 配置文件 ​

创建 config.yaml：

yaml
server:
  port: 8090
  mode: release

database:
  driver: sqlite
  database: ./data/vcenter.db

auth:
  jwt_secret: "your-secure-secret-key"

logging:
  level: info
  format: json

3. 启动服务 ​

bash
./vcenter

配置说明 ​

配置项环境变量默认值说明
server.portVCENTER_PORT8090服务端口
server.modeVCENTER_MODEdebug运行模式
database.driverVCENTER_DB_DRIVERsqlite数据库驱动
database.hostVCENTER_DB_HOSTlocalhost数据库主机
auth.jwt_secretVCENTER_JWT_SECRET-JWT 密钥

升级 ​

bash
cd /opt/vcenter
docker compose pull
docker compose up -d

备份与恢复 ​

备份 ​

bash
# 备份数据目录
tar -czf vcenter-backup-$(date +%Y%m%d).tar.gz /opt/vcenter/data

恢复 ​

bash
# 停止服务
docker compose down

# 恢复数据
tar -xzf vcenter-backup-20231201.tar.gz -C /

# 启动服务
docker compose up -d

常见问题 ​

端口被占用 ​

bash
# 查看端口占用
lsof -i :8090

# 修改配置使用其他端口
# 编辑 docker-compose.yml，将 8090 改为其他端口

无法连接数据库 ​

检查数据目录权限：

bash
chown -R 1000:1000 /opt/vcenter/data
