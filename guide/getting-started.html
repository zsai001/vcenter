快速开始 ​

本指南将帮助你在 5 分钟内启动 VCenter。

环境要求 ​

  • Linux 服务器（推荐 Ubuntu 22.04 / CentOS 8+）
  • 2 核 CPU / 4GB 内存（最低配置）
  • Go 1.24+ (开发环境) 或 Docker

方式一：Docker 部署 ​

1. 下载安装脚本 ​

bash
curl -sSL https://vcenter.zsoft.cc/install.sh | bash

2. 启动服务 ​

bash
cd /opt/vcenter
docker compose up -d

3. 访问控制台 ​

打开浏览器访问 http://your-server-ip:8090

默认管理员账户：

  • 用户名：admin
  • 密码：admin123

安全提示

请在首次登录后立即修改默认密码！

方式二：开发环境 ​

1. 克隆代码 ​

bash
git clone https://github.com/zsai001/vcenter_src.git
cd vcenter_src

2. 启动后端 ​

bash
cd center
go mod tidy
go run ./cmd/server

3. 使用开发脚本 ​

bash
chmod +x dev.sh
./dev.sh dev

添加节点 ​

VCenter 通过 Agent 管理 VPanel 节点：

  1. 登录 VCenter 控制台
  2. 进入「节点管理」
  3. 点击「添加节点」
  4. 复制安装脚本到目标服务器执行
  5. Agent 自动连接到 VCenter
bash
# 在目标服务器上执行
curl -sSL https://your-vcenter/install-agent.sh | bash -s -- --token=xxx

添加 Kubernetes 集群 ​

  1. 登录 VCenter 控制台
  2. 进入「Kubernetes」-「集群管理」
  3. 点击「添加集群」
  4. 上传 kubeconfig 文件或填写连接信息
  5. 点击「验证并保存」

支持的认证方式：

  • Kubeconfig - 上传 kubeconfig 文件
  • Token - Service Account Token
  • 证书 - 客户端证书认证

资源管理 ​

添加节点或集群后，VCenter 会自动同步资源：

节点资源 ​

  • 实时系统状态
  • CPU / 内存 / 磁盘指标
  • Docker 容器
  • 系统服务

Kubernetes 资源 ​

  • Deployments / StatefulSets / DaemonSets
  • Services / Ingresses
  • Pods (日志 / 终端)
  • ConfigMaps / Secrets
  • PersistentVolumes
