部署指南 ​

本文档详细介绍 VCenter 在生产环境中的部署方式。

部署方式 ​

VCenter 支持多种部署方式：

方式适用场景复杂度
二进制部署小规模、快速测试⭐
Docker 部署单机生产环境⭐⭐
Docker Compose完整单机部署⭐⭐
Kubernetes高可用生产环境⭐⭐⭐

二进制部署 ​

1. 下载二进制 ​

bash
# 从 GitHub Release 下载
curl -LO https://github.com/zsai001/vcenter/releases/latest/download/vcenter-linux-amd64.tar.gz

# 解压
tar -xzf vcenter-linux-amd64.tar.gz
cd vcenter

2. 配置文件 ​

创建 config.yaml：

yaml
server:
  port: 8090
  mode: release
  web_dir: ./web

database:
  driver: sqlite
  database: ./data/vcenter.db

auth:
  jwt_secret: "your-secret-key-change-in-production"
  token_expiry: 60

logging:
  level: info
  format: json
  output: stdout

3. 启动服务 ​

bash
# 直接启动
./vcenter

# 后台运行
nohup ./vcenter > /var/log/vcenter.log 2>&1 &

4. Systemd 服务 ​

创建 /etc/systemd/system/vcenter.service：

ini
[Unit]
Description=VCenter Server
After=network.target

[Service]
Type=simple
User=vcenter
WorkingDirectory=/opt/vcenter
ExecStart=/opt/vcenter/vcenter
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

启用服务：

bash
sudo systemctl daemon-reload
sudo systemctl enable vcenter
sudo systemctl start vcenter

Docker 部署 ​

单容器部署 ​

bash
docker run -d \
  --name vcenter \
  -p 8090:8090 \
  -v vcenter_data:/app/data \
  -e VCENTER_MODE=release \
  -e VCENTER_JWT_SECRET=your-secret-key \
  vcenter/server:latest

Docker Compose ​

创建 docker-compose.yml：

yaml
version: '3.8'

services:
  vcenter:
    image: vcenter/server:latest
    container_name: vcenter
    restart: always
    ports:
      - "8090:8090"
    environment:
      - VCENTER_MODE=release
      - VCENTER_DB_DRIVER=postgres
      - VCENTER_DB_HOST=postgres
      - VCENTER_DB_PORT=5432
      - VCENTER_DB_NAME=vcenter
      - VCENTER_DB_USER=vcenter
      - VCENTER_DB_PASSWORD=${DB_PASSWORD}
      - VCENTER_JWT_SECRET=${JWT_SECRET}
    volumes:
      - vcenter_data:/app/data
    depends_on:
      - postgres
    networks:
      - vcenter-network

  postgres:
    image: postgres:16-alpine
    container_name: vcenter-postgres
    restart: always
    environment:
      - POSTGRES_DB=vcenter
      - POSTGRES_USER=vcenter
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - vcenter-network

volumes:
  vcenter_data:
  postgres_data:

networks:
  vcenter-network:
    driver: bridge

创建 .env 文件：

bash
DB_PASSWORD=your-secure-db-password
JWT_SECRET=your-secure-jwt-secret

启动：

bash
docker compose up -d

Nginx 负载均衡配置 ​

nginx
upstream vcenter {
    least_conn;
    server vcenter1:8090 weight=1;
    server vcenter2:8090 weight=1;
    server vcenter3:8090 weight=1;
}

server {
    listen 80;
    server_name vcenter.example.com;

    location / {
        proxy_pass http://vcenter;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /ws {
        proxy_pass http://vcenter;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

HTTPS 配置 ​

使用 Let's Encrypt ​

bash
# 安装 certbot
apt install certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d vcenter.example.com

# 自动续期
certbot renew --dry-run

手动 TLS 配置 ​

在 config.yaml 中启用 TLS：

yaml
server:
  port: 8090
  tls:
    enabled: true
    cert_file: /etc/ssl/vcenter/cert.pem
    key_file: /etc/ssl/vcenter/key.pem

数据库配置 ​

PostgreSQL 配置 ​

yaml
database:
  driver: postgres
  host: localhost
  port: 5432
  database: vcenter
  username: vcenter
  password: your-password
  ssl_mode: require

数据库初始化 ​

sql
-- 创建用户和数据库
CREATE USER vcenter WITH PASSWORD 'your-password';
CREATE DATABASE vcenter OWNER vcenter;
GRANT ALL PRIVILEGES ON DATABASE vcenter TO vcenter;

备份与恢复 ​

数据备份 ​

bash
# PostgreSQL 备份
pg_dump -h localhost -U vcenter -d vcenter > backup.sql

# SQLite 备份
sqlite3 vcenter.db ".backup 'backup.db'"

定时备份脚本 ​

bash
#!/bin/bash
# /opt/vcenter/backup.sh

BACKUP_DIR=/opt/vcenter/backups
DATE=$(date +%Y%m%d_%H%M%S)

pg_dump -h localhost -U vcenter -d vcenter > $BACKUP_DIR/vcenter_$DATE.sql

# 保留最近 7 天的备份
find $BACKUP_DIR -name "vcenter_*.sql" -mtime +7 -delete

添加 crontab：

bash
# 每天凌晨 2 点备份
0 2 * * * /opt/vcenter/backup.sh

监控与日志 ​

日志配置 ​

yaml
logging:
  level: info      # debug, info, warn, error
  format: json     # json, console
  output: stdout   # stdout, file

日志轮转 ​

使用 logrotate：

/var/log/vcenter/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
}
